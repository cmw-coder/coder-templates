# 当前文件包含生成测试脚本需要关注的注意事项

# 注意点1：
禁止在test_step步骤中使用gl.DUTX.send中下发命令

# 注意点2：
在类中调用方法occurrences应该使用TestClass.occurrences,调用其他方法同理

# 注意点3：
使用remove和delete方法时需要同时针对非索引列进行删除操作

# 注意点4：
`Netconf.txt`和`yin.txt`描述的是同一张表，其中`待测yin.txt`文件描述更准确，若两者不同，以`待测yin.txt`文件描述为准，禁止修改元素名称

# 注意点5：
对于可配置的变量需要遵守`yin.txt`文件中的数值范围，并且要结合前面的测试步骤选择合理的参数值，当参数值和前置的测试步骤参数有关时，要确定这些参数是否能正常下发的，对于一些参数范围需要构造特殊符号下发的情况，特殊符号避免&、>、<、?这四个特殊符号。

部分元素间的取值关系可能存在对应关系，注意看yin.txt介绍和Netconf.txt

# 注意点6：
`Netconf.txt`文件中Elements标注的default可变量表示缺省值或默认值，若配置成默认值，get-config类方法获取时不显示。
`yin.txt`文件中<default>标签表示缺省值或默认值(举例说明<default value="1"/>，说明默认值是1)，若配置成默认值，get-config类方法获取时不显示。
当'netconf.txt'中默认值和'yin.txt'同时存在默认值，以yin.txt中的默认值为标准默认值


# 注意点7:
若表中元素配置成默认值，对于delete方法不支持删除，若某个可配置变量配置了非默认值delete可删除，remove方法支持删除任意值包括元素的默认值

# 注意点8：
在编写脚本后按照Checklist.json文件内容进行检查并修改

# 注意点9：

当测试步骤测试参数长度相关时，需要你注意参数的长度。

# 注意点10：

Can not edit tables with only table names.
